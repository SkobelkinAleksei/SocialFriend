# SocialFriend

Пет-проект социальной сети, построенный на микросервисной архитектуре. Основной упор сделан на разделение бизнес-логики, межсервисное взаимодействие и асинхронность.

## Технологический стек
* **Язык:** Java 21
* **Фреймворк:** Spring Boot , Spring Cloud (Gateway), Hibernate 
* **Базы данных:** PostgreSQL (для каждого сервиса своя БД)
* **Брокер сообщений:** Apache Kafka (асинхронная доставка уведомлений)
* **Сборка:** Gradle (многомодульный проект)
* **Инструменты:** MapStruct, Lombok, RestTemplate

## Архитектура
Проект разделен на независимые модули:
* **Gateway (8080):** Единая точка входа, маршрутизация запросов.
* **Security/Auth:** Регистрация и авторизация JWT. Синхронизация данных пользователей через Kafka.
* **Friend Service:** Управление заявками и связями между пользователями.
* **Post Service:** Создание контента, управление доступом к комментариям.
* **Comment Service:** Система комментариев с проверкой прав через межсервисные вызовы.
* **Like Service:** Система реакций.
* **Notification Service:** Асинхронный сервис, собирающий события из системы через Kafka.
* **Common & Rest-Client:** Общие библиотеки для DTO, Enum и унифицированной работы с HTTP-запросами.

## Ключевые особенности
* **Асинхронность:** Уведомления о лайках, комментариях и заявках в друзья отправляются через Kafka, что не блокирует основной поток выполнения бизнес-логики.
* **Слабая связность:** Сервисы общаются через REST синхронно только там, где это необходимо для валидации, в остальных случаях используется событийная модель.
* **Безопасность:** Реализован проброс заголовков авторизации и идентификатора пользователя (`X-User-Id`)
* **Чистый код(в разработке):** Использование паттернов проектирования (Facade(нужно реализовать), Global Exception Handling).
